# Maintenance & Troubleshooting

## Monitoring

### Key Metrics
-   **Pub/Sub Topic Detail**: Check "Publish message operations" and "Unacked message count".
-   **Load Balancer**: Check "Frontend RTT" and "Backend Latency".

## Common Issues

### "Certificate Expired"
Self-signed certificates generated by `./generate_certs.sh` are valid for **365 days**.
-   **Fix**: Run `./generate_certs.sh` again and redeploy (or update the Secret Manager if used).

### "Permission Denied" (Artifact Registry)
-   **Error**: `denied: gcr.io repo does not exist` or `permission denied`.
-   **Fix**: Ensure `artifactregistry.googleapis.com` API is enabled and the service account has `artifactregistry.writer`. Running `./init` re-applies these bindings.

### "terraform.tfvars not found"
-   **Fix**: Run `./generate_certs.sh` to generate a fresh `terraform.tfvars` with self-signed certs, or manually create the file with your production certificates.

### "Unhealthy MIG / Instance"
If the Load Balancer health check fails:
1.  **SSH into the instance**: `gcloud compute ssh <instance-name>`
2.  **Check Startup Script**: `sudo journalctl -u google-startup-scripts.service -n 100`
    -   *Common Error*: `docker-credential-gcr: command not found` -> Update `compute.tf` to use `gcloud auth configure-docker`.
    -   *Common Error*: `invalid reference format` -> Ensure `bridge_image` variable is correctly passed to Terraform.
3.  **Check Containers**: `sudo docker ps`
    -   If `mosquitto` is running but `bridge` is not, check `sudo docker logs <bridge_container_id>`.

### "SSL Error: Certificate Verify Failed"
When using `mosquitto_pub` with self-signed certificates:
-   **Cause**: You are connecting via IP (`34.x.x.x`) but the cert is for a domain (`mqtt.example.com`).
-   **Fix 1 (Test)**: Use `--insecure` to bypass hostname verification (but still valid CA).
    ```bash
    mosquitto_pub -h <LB_IP> -p 8883 --cafile certs/ca.pem --insecure ...
    ```
-   **Fix 2 (Prod)**: Configure DNS or `/etc/hosts` to map the LB IP to the domain name.
